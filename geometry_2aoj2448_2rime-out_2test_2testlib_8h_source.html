434 /*
435  * Very simple regex-like pattern.
436  * It used for two purposes: validation and generation.
437  *
438  * For example, pattern("[a-z]{1,5}").next(rnd) will return
439  * random string from lowercase latin letters with length
440  * from 1 to 5. It is easier to call rnd.next("[a-z]{1,5}")
441  * for the same effect.
442  *
443  * Another samples:
444  * "mike|john" will generate (match) "mike" or "john";
445  * "-?[1-9][0-9]{0,3}" will generate (match) non-zero integers from -9999 to 9999;
446  * "id-([ac]|b{2})" will generate (match) "id-a", "id-bb", "id-c";
447  * "[^0-9]*" will match sequences (empty or non-empty) without digits, you can't
448  * use it for generations.
449  *
450  * You can't use pattern for generation if it contains meta-symbol '*'. Also it
451  * is not recommended to use it for char-sets with meta-symbol '^' like [^a-z].
452  *
453  * For matching very simple greedy algorithm is used. For example, pattern
454  * "[0-9]?1" will not match "1", because of greedy nature of matching.
455  * Alternations (meta-symbols "|") are processed with brute-force algorithm, so
456  * do not use many alternations in one expression.
457  *
458  * If you want to use one expression many times it is better to compile it into
459  * a single pattern like "pattern p("[a-z]+")". Later you can use
460  * "p.matches(std::string s)" or "p.next(random_t& rd)" to check matching or generate
461  * new string by pattern.
462  *
463  * Simpler way to read token and check it for pattern matching is "inf.readToken("[a-z]+")".
464  */
